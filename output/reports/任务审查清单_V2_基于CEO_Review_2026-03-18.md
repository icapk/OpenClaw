# 任务审查清单 V2（基于 CEO Review）

> 目的：把“高标准思维”变成执行前强制检查，减少超时与返工。

## 0. 模式锁定（先选一种，不可中途漂移）
- [ ] 范围扩展（追求10x价值）
- [ ] 范围保持（既定范围防弹交付）
- [ ] 范围缩减（最小可交付）

## 1. 前提挑战
- [ ] 这是正确问题吗？
- [ ] 目标用户/业务结果是什么？
- [ ] 什么都不做会怎样？

## 2. 依赖与路径
- [ ] 关键依赖已锁定负责人与时间
- [ ] 数据流4路径已覆盖：正常 / nil / 空输入 / 上游错误
- [ ] 关键交互边界已覆盖：双击、返回、离开、慢网、超时

## 3. 错误与可观测
- [ ] 无静默失败（Zero Silent Failures）
- [ ] 每类错误有名字、触发条件、用户可见反馈、处理动作
- [ ] 日志/指标/告警/Runbook 已纳入交付范围

## 4. 验收与门禁
- [ ] DoD 可验证，验收口径前置
- [ ] 关单证据模板字段全部可提交
- [ ] 关键回归项已列入门禁（失败即打回）

## 5. 时间与升级
- [ ] SLA 已明确：首轮、终版、升级阈值
- [ ] T-24h/T+0/T+120% 升级路径已定义
- [ ] 变更控制（范围/DDL）已定义并可追溯

## 6. 输出要求（关单前）
- [ ] 结论（通过/不通过）
- [ ] 缺陷分级与复现步骤
- [ ] 证据路径（日志/截图/批次时间戳）
- [ ] 风险残留与后续计划

---

**执行规则：**任一关键项缺失，不得进入执行/不得关单。
