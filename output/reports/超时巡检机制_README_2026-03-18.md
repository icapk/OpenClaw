# 超时巡检机制说明（2026-03-18）

## 目标
定期扫描 `tasks/in_progress` 下的任务文件，识别超过 24 小时未更新的任务，并产出可追踪的巡检报告。

## 脚本位置
- `/Users/a1/.openclaw/workspace/scripts/overdue_audit.py`

## 扫描规则
- 扫描目录：`/Users/a1/.openclaw/workspace/tasks/in_progress/*.md`
- 判断标准：文件最后修改时间（mtime）距当前时间超过 `24` 小时

## 输出
1. **Markdown 报告**
   - 目录：`/Users/a1/.openclaw/workspace/output/reports/`
   - 文件名：`超时巡检_YYYY-MM-DD_HHMM.md`
   - 内容包含：巡检时间、规则、统计、超时明细、全量文件列表

2. **标准输出（stdout）**
   - 总数
   - 超时数
   - 文件名列表（超时文件带 `[超时]` 标记）

## 使用方法
```bash
python3 /Users/a1/.openclaw/workspace/scripts/overdue_audit.py
```

## 可选参数
```bash
python3 /Users/a1/.openclaw/workspace/scripts/overdue_audit.py \
  --tasks-dir /Users/a1/.openclaw/workspace/tasks/in_progress \
  --report-dir /Users/a1/.openclaw/workspace/output/reports \
  --threshold-hours 24
```

## 退出码
- `0`：执行成功

## 建议
- 可将脚本加入定时任务（如 crontab）实现自动化巡检。
- 若后续需要扩展，可增加“按负责人分组”或“推送消息通知”能力。
