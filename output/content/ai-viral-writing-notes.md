## 一、7个神级技巧学习笔记

### 技巧1：爆款标题生成

**核心公式：数字 + 痛点 + 解决方案**

**学习理解：**
- 数字制造"确定性"和"可预期感"，降低读者决策门槛
- 痛点是情绪触发器，让人感觉"这说的是我"
- 解决方案是"承诺"，让读者知道点进来能得到什么
- 三者合一 = 点击率保障

**操作流程（新增重点）：**
> 让AI生成20个标题 → 用指令筛选最有悬念感的3个 → 优中选优

这个"20选3"的批量生成+筛选机制非常关键，解决了"想不出标题"的创作瓶颈。

**对我们现有文章的改进建议：**
- 每次写文章前，用AI批量生成20个标题（数字型、痛点型、悬念型各若干）
- 设定筛选标准：悬念感、情绪张力、身份识别度
- 对比现有标题打分，低于8分的必须重写

**改进我们现有文章的具体方案：**
```
原文标题："AI Agent 协作：内容即产品：自动化增长实验"
问题：太学术、太长、没有情绪

→ 批量生成20个标题后筛选：
✅ "手工写内容1天，AI只要10分钟：内容团队都在用的自动化流"
✅ "内容团队最该淘汰的环节，不是编辑"
✅ "3步把内容产能翻2.3倍：我们实测有效的方法"
```

### 技巧2：选题灵感库

**核心：用AI建立爆款选题池，自动监控竞品账号，分析100篇文章总结选题规律**

**学习理解：**
- 选题是爆款的起点，选题不对，再好的内容也没人看
- 用AI监控竞品账号 = 低成本的爆款规律发现
- 分析100篇文章 = 用数据说话，总结规律而非凭感觉

**对我们现有选题的建议：**
- 每周固定时间用AI扫描3-5个对标账号的爆款文章
- 记录：标题、阅读量、发布时间、选题角度
- 建立选题池文档，按"痛点类型"分类

**改进我们现有文章的具体方案：**
```
我们现有选题来源比较单一（主要是技术视角）
→ 增加3个选题维度：
1. 老板/管理者视角：内容团队怎么管、怎么考核
2. 打工人视角：怎么做内容效率更高、少加班
3. 行业观察视角：竞品动态、行业趋势背后的内容逻辑
```

### 技巧3：文章结构设计

**核心：用框架思维拆解爆款，四种经典结构：**

| 结构类型 | 适用场景 | 爆款特征 |
|---------|---------|---------|
| **痛点引入式** | 观点类、干货类 | 先说痛点，再说方案，情绪共鸣强 |
| **清单干货式** | 教程类、攻略类 | 数字列表，承诺明确，阅读门槛低 |
| **故事叙事式** | 案例类、人物类 | 有代入感，适合讲完整逻辑 |
| **对比反差式** | 评测类、观点类 | 利用反差制造冲突，吸引点击 |

**学习理解：**
- 爆款文章不是"写出来的"，是"设计出来的"
- 动笔前先选结构，就是选读者阅读路径
- 四种结构可以组合使用（比如开头用痛点引入，中间用清单干货）

**对我们现有文章结构的改进建议：**
- 写之前先确定：我要用哪种结构？
- 避免"想到什么写什么"的自由散漫式写作
- 每篇文章结构类型要单一，不要混搭

**改进我们现有文章的具体方案：**
```
现有文章《AI Agent 协作》用的是"功能介绍式"结构
→ 改成"痛点引入式 + 清单干货式"组合：
- 开头：内容团队的3个至暗时刻（痛点共鸣）
- 中间：3步解决方案（清单干货）
- 结尾：互动提问 + 转发引导
```

### 技巧4：开头写作

**核心：3秒抓住读者注意力，四种开头类型：**

| 开头类型 | 第一句话特点 | 适合话题 |
|---------|------------|---------|
| **痛点共鸣型** | "你是不是也这样……" | 通用型，万能开头 |
| **数据冲击型** | "XX%的XX人都……" | 数据敏感话题 |
| **故事悬念型** | 讲一个未完成的故事 | 有案例可说 |
| **权威背书型** | "XX大咖都在用……" | 工具推荐类 |

**学习理解：**
- 开头是全文最重要的部分，决定读者是否继续
- 3秒内必须让读者产生"说中了！"或"然后呢？"的感觉
- 不要在开头解释"我要说什么"，直接说

**对我们现有文章开头的改进建议：**
- 删掉所有"本文将从以下几个方面展开"式的开头
- 第1句话必须是痛点/数字/故事/冲突之一
- 开头不超过100字，必须在100字内制造"钩子"

**改进我们现有文章的具体方案：**
```
原文开头：
"## 一、为什么这个话题值得做
这个选题热度 95，正处于增长窗口。"

→ 改成痛点共鸣型开头：
"周一早上，打开飞书，12个待写选题在排队。
开会又被催：'今天的文章呢？'
这不是你们不够努力，是内容生产的第一逻辑就错了——
我们一直把内容当'创意活儿'，却忘了它首先是个'系统工程'。"
```

### 技巧5：内容生成

**核心：分段式写作，不要一次性让AI写完整篇**

**学习理解：**
- AI一次性写完整篇，容易"跑偏"或"水"
- 分段给明确指令 = 每段都是定向输出，质量可控
- 明确指令包括：这段要说什么、情绪是什么、字数多少、有无金句要求

**操作流程：**
```
第一步：给AI整体框架（标题+结构+每段核心观点）
第二步：分段生成开头（写完审核再继续）
第三步：分段生成每个模块（审核通过再继续）
第四步：分段生成结尾
第五步：整体润色
```

**对我们现有写作流程的改进建议：**
- 取消"AI一次性生成3000字"的操作
- 改为"指令+审核+继续"的分段模式
- 每段生成后人工审核：是否跑题、是否空洞、情绪是否连贯

**改进我们现有文章的具体方案：**
```
原文写作流程：
用户 → 一次给主题 → AI生成3000字全文

→ 新的分段写作流程：
1. 先给AI文章框架（8-10个核心点）
2. 逐段生成，每段200-300字
3. 每段审核：情绪是否连贯、有无废话
4. 通过后再生成下一段
5. 结尾单独生成，单独润色
```

### 技巧6：排版美化

**核心：让AI生成可直接复制的HTML，只用 section + p + span 标签，全部行内样式**

**学习理解：**
- 公众号排版直接影响阅读体验和读完率
- HTML格式 = 可直接复制到公众号编辑器，无需手动排版
- 只用 section + p + span = 结构清晰，不乱码
- 全部行内样式 = 各平台兼容性最好

**对我们现有排版的改进建议：**
- 以后让AI生成内容时，直接要求"生成可直接复制的HTML排版"
- 指定标签类型：section做段落组，p做正文段落，span做强调
- 行内样式统一：用font-size/color/font-weight控制层级

**改进我们现有文章的具体方案：**
```
原文排版：纯Markdown，普通编辑器手动调整

→ 新排版流程：
1. 内容生成完成后，让AI输出HTML版本
2. 指定格式：section（模块）+ p（段落）+ span（强调）
3. 颜色规范：正文#333333，小标题#1a1a1a，强调#E54D00
4. 直接复制HTML到公众号，无痛排版
```

### 技巧7：结尾设计

**核心：提高互动率和转化率，四件套：**

| 结尾组件 | 作用 | 示例 |
|---------|------|------|
| **金句总结** | 强化记忆，让人想截图 | "内容不是灵感游戏，而是系统工程" |
| **互动提问** | 提高评论率，引发讨论 | "你们团队现在卡在哪？评论区说说" |
| **福利钩子** | 提高关注/转发率 | "想要这套模板？私信'666'领取" |
| **推荐阅读** | 增加点击深度，提升站内阅读时长 | "延伸阅读：怎么用这套方法做选题" |

**学习理解：**
- 结尾是最后的"转化机会"，错过就没有了
- 四件套至少用2件，效果最好
- 互动提问要具体，"你们怎么看"太虚，"你们的卡点是XX还是YY"才有人回答

**对我们现有文章结尾的改进建议：**
- 所有文章结尾必须包含：金句总结 + 至少1个互动提问
- 福利钩子和推荐阅读按内容类型选用

**改进我们现有文章的具体方案：**
```
原文结尾：
"内容不是灵感游戏，而是可运营的系统工程。"

→ 改成四件套结尾：

【金句总结】
内容不是灵感游戏，而是一套可优化的系统工程。

【互动提问】
你们团队现在内容生产最大的卡点是什么？
A. 选题难 B. 写作慢 C. 复盘累 D. 全选

【福利钩子】
想要这套自动化内容模板（飞书文档版）？
评论区扣"想要"，我私信发你。

【推荐阅读】
→ 延伸阅读：《从0到1搭建内容团队：3个阶段的关键动作》
```

## 二、7个技巧的整合应用：AI辅助写作标准流程

结合7个技巧，形成一套完整的AI辅助写作流程：

```
第一步：选题（第1-2分钟）
  → 用AI扫描竞品爆款，建立本周选题池
  → 从选题池选1个最适合自己的

第二步：标题（第3-5分钟）
  → 用公式生成20个标题
  → 筛选最有悬念感的3个
  → 选最优1个

第三步：结构（第2分钟）
  → 选定文章结构（痛点引入/清单干货/故事/对比）
  → 设计文章大纲（8-10个核心点）

第四步：开头（第5分钟）
  → 选定开头类型（痛点/数据/故事/权威）
  → 写3个版本，优选1个

第五步：正文分段生成（核心环节）
  → 分段给AI明确指令
  → 每段审核后再继续
  → 避免一次生成全文

第六步：结尾（第3分钟）
  → 金句总结 + 互动提问 + 福利钩子（选配）

第七步：排版（第2分钟）
  → 让AI输出HTML版本
  → 直接复制到公众号编辑器

第八步：发布前检查
  → 标题是否够悬念？
  → 开头3秒是否有钩子？
  → 正文是否有案例/故事？
  → 结尾是否有互动引导？
```

## 五、关键认知升级

> **从"写好文章"到"设计好文章"**

新学的7个技巧最核心的转变是：
- 爆款文章不是"写出来的"，是**设计出来的**
- 每一个环节（选题→标题→结构→开头→正文→排版→结尾）都有可以优化的点
- AI是工具，不是替代者——用它批量生成，但人来筛选和决策
