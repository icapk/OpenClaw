## 一、TrendRadar 核心功能理解

### 1.1 工具定位
TrendRadar 是一款 **AI 驱动的舆情监控与热点追踪工具**，核心理念是"告别信息过载，AI 助你看懂热点"。

### 1.2 核心功能清单

| 功能模块 | 具体能力 |
|---------|---------|
| **多平台热点聚合** | 抖音、知乎、B站、今日头条、百度热搜、微博、微信等 |
| **RSS 订阅** | 支持自定义 RSS 源，扩展内容来源 |
| **关键词精准筛选** | 用户定义关键词组合，只推送高度相关内容 |
| **AI 智能筛选** | 基于 MCP 协议，自动过滤无关信息，保留核心内容 |
| **AI 翻译与摘要** | 支持多语言内容翻译，提供 AI 生成的简报摘要 |
| **多渠道推送** | 企业微信、飞书、钉钉、邮件、Telegram 等 |
| **本地数据存储** | 数据本地保存，可回溯、可复用 |
| **Docker 部署** | 一键部署，零门槛，GitHub Fork 方案免环境配置 |
| **趋势分析** | 多维度趋势追踪、情感分析、相似内容检索 |

### 1.3 核心价值总结
- **主动性**：由工具主动推送热点，不依赖人工刷屏
- **精准性**：关键词 + AI 双重过滤，减少无效信息
- **聚合性**：一屏看全网，无需切换多个平台
- **可扩展性**：支持 RSS 接入竞品平台，支持 MCP 接入更强 AI 能力

## 二、选题池建立方案

### 2.1 选题池是什么
选题池是 **待写文章的候选主题库**，是持续产出的弹药库。传统做法是临时找灵感，效率低且质量不稳；用 TrendRadar 可以 **主动囤积、持续积累、随时调用**。

### 2.2 用 TrendRadar 建立选题池的三层结构

```
选题池三层结构
├── 🔥 热点层（TrendRadar 实时热点）
│   ├── 来源：各平台热搜榜
│   ├── 用途：追时效热点，快速产出
│   └── 保留周期：7天内有效
│
├── 📚 主题层（TrendRadar 关键词监控）
│   ├── 来源：设置领域关键词（如：AI、职场、成长）
│   ├── 用途：深耕垂直领域，积累素材
│   └── 保留周期：长期有效
│
└── 🔍 竞品层（RSS 订阅竞品账号）
    ├── 来源：竞品公众号、头部账号 RSS
    ├── 用途：拆解竞品爆款，找切入角度
    └── 保留周期：长期有效
```

### 2.3 具体实施步骤

**Step 1：配置热点聚合源**
- 开启抖音、知乎、B站、微博、百度热搜、今日头条
- 评估各平台在我们定位领域的热点频率，优先关注高频平台

**Step 2：设置关键词矩阵**
- 核心关键词（必须监控）：如「AI 工具」「效率提升」「职场成长」「副业赚钱」
- 长尾关键词（精准选题）：如「ChatGPT 写文案」「Notion 模版」「小红书运营」
- 品牌关键词：监控自身品牌和竞品品牌动态

**Step 3：接入竞品 RSS**
- 收集同领域头部公众号的 RSS 源（通过 RSSHub 等服务转换）
- 用 TrendRadar 的 RSS 功能统一监控
- 发现竞品爆款文章后，分析其选题角度

**Step 4：选题分级入库**
- 热点新闻 → 标注「🔥热点」，优先快速产出
- 深度主题 → 标注「📚主题」，积累素材后产出
- 竞品爆款 → 标注「🔍竞品」，找差异化切入角度

## 三、热点监控流程

### 3.1 每日热点监控流程

```
┌─────────────────────────────────────────────────────────┐
│                    每日热点监控 SOP                       │
└─────────────────────────────────────────────────────────┘

     ┌──────────────┐
     │  早会/开工   │  9:00 打开 TrendRadar 推送消息
     └──────┬───────┘
            ▼
     ┌──────────────┐
     │  第一轮筛选   │  过滤与定位无关内容，保留 5-10 条高相关热点
     └──────┬───────┘
            ▼
     ┌──────────────┐
     │  选题评估    │  每条热点问三个问题：  ① 和我的定位相关吗？  ② 有独特角度吗？  ③ 能写出价值吗？
     └──────┬───────┘
            ▼
     ┌──────────────┐
     │  入库/放弃   │  合格 → 写入选题池待写队列  不合格 → 归档备注原因
     └──────┬───────┘
            ▼
     ┌──────────────┐
     │  竞品扫描    │  查看竞品 RSS，看是否有值得拆解的爆款
     └──────┬───────┘
            ▼
     ┌──────────────┐
     │  下班前汇总  │  整理当日选题池，发送到飞书群存档
     └──────────────┘
```

### 3.2 热点分类处理策略

| 热点类型 | 处理策略 | 产出要求 |
|---------|---------|---------|
| **突发新闻** | 快速响应，2 小时内出稿 | 快、准、简短，配观点 |
| **持续发酵热点** | 深度分析，第 2-3 天出稿 | 有深度、有独到见解 |
| **领域日常热点** | 结合自身定位，判断是否跟进 | 找角度，避免同质化 |
| **长期趋势热点** | 积累素材，系统性输出 | 做成系列文章 |

### 3.3 推送渠道配置（推荐飞书）

TrendRadar 支持飞书 Webhook，可配置推送到我们的飞书运营群：

```yaml
# TrendRadar 推送配置示例
push_channels:
  - feishu:
      webhook_url: "https://open.feishu.cn/open-apis/bot/v2/hook/xxx"
      keyword_filter: ["AI", "效率", "职场", "工具"]
      daily_digest: true   # 每日摘要
      realtime_alert: true # 关键词命中实时告警
```

### 3.4 选题评估维度（AI 辅助）

利用 TrendRadar 的 AI 分析能力，对热点进行自动评分：

| 评估维度 | 权重 | 说明 |
|---------|------|------|
| **热度指数** | 20% | 全网讨论量、增长速度 |
| **相关度** | 30% | 与账号定位的匹配程度 |
| **可切入角度** | 25% | 是否有独特视角，避免同质化 |
| **时效窗口** | 15% | 热点生命周期，还来得及吗 |
| **价值深度** | 10% | 能给读者带来什么实际价值 |

## 四、竞品监控专项方案

### 4.1 竞品监控目标
1. **发现爆款规律**：什么话题容易爆，爆款有什么共同特征
2. **找差异化角度**：竞品写了什么，我们怎么写出不同
3. **追踪行业动态**：竞品在推什么方向，我们跟上或超越

### 4.2 竞品 RSS 接入方式

通过 RSSHub 将公众号转为 RSS：

```
竞品公众号 → RSSHub → TrendRadar RSS 模块 → 统一聚合
```

推荐监控的竞品维度：
- **同类账号**：同领域、同受众的头部账号
- **跨类账号**：不同领域但受众重叠的账号（找灵感）
- **标杆账号**：行业内最顶尖的账号（学习结构）

### 4.3 竞品爆款分析流程

```
发现竞品爆款文章
       ↓
用 TrendRadar 记录标题、阅读量、发布时间
       ↓
分析爆款要素：
  - 选题：为什么这个话题会爆？
  - 角度：竞品从什么角度切入？
  - 标题：用了什么关键词/句式？
  - 开头：怎么吸引人点进来？
       ↓
决定是否跟写 → 找差异化角度 → 入选题池
```

## 六、注意事项

1. **热点不等于选题**：TrendRadar 推送的是热点，最终是否成文需要人工判断
2. **避免同质化**：热门话题很多人写，要有独特切入角度才跟进
3. **时效性管理**：热点有窗口期，突发热点要快速响应，不要拖延
4. **质量优先**：TrendRadar 是弹药库，但文章质量才是核心竞争力
