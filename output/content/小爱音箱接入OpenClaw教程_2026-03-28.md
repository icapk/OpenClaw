## 🧰 准备工作（10分钟搞定）

### 硬件要求

- ✅ 一台小爱智能音箱（任意型号都行）
- ✅ 米家APP已绑定该音箱（设备名称要记住，后面要用）
- ✅ 一台电脑/服务器（用于运行OpenClaw）

### 软件准备

1. **OpenClaw Gateway**（主程序）
   - 还没装？看官方文档安装，5分钟搞定
   
2. **migpt-claw插件**（小爱专用插件）
   - 项目地址：https://github.com/yuanlong/migpt-claw
   - 安装包：`migpt-claw-1.0.0.tgz`

3. **小米账号信息**（重点来了！）

获取方式：

```
1. 打开电脑浏览器，访问 account.xiaomi.com
2. 按 F12 打开开发者工具
3. 切到 Network（网络）标签
4. 登录小米账号
5. 在网络请求里找 passToken 字段——以 V1 开头的就是长期令牌
6. userId 是你的小米数字ID，也在返回的响应里
```

⚠️ **注意**：password 就是你的小米账号密码，简单粗暴。

## 🚀 详细步骤（手把手）

### Step 1：安装插件

在终端运行：

```bash
openclaw plugins install ./migpt-claw-1.0.0.tgz
```

安装完成后，插件就准备好了。

### Step 2：配置插件

打开配置文件（没有就新建）：

```bash
nano ~/.openclaw/openclaw.json
```

在 `plugins` 数组里加入 `migpt-claw`，然后在根级添加以下配置：

```json
{
  "plugins": ["migpt-claw"],
  "migpt": {
    "userId": "你的小米数字ID",
    "password": "你的小米密码",
    "passToken": "V1开头的长期令牌",
    "devices": "你的音箱名称",
    "announceOnStart": true,
    "startupMessage": "您的小龙虾已上线，随时为您服务",
    "acknowledgeOnReceive": true,
    "receiveMessage": "收到！"
  }
}
```

📝 **配置说明**：

| 参数 | 含义 | 示例 |
|------|------|------|
| `userId` | 小米数字ID | 1234567890 |
| `password` | 小米密码 | MiPassword123 |
| `passToken` | V1开头的长期令牌 | V1~xxxxx |
| `devices` | 音箱名称（须和米家APP一致） | "小爱音箱Pro" |
| `startupMessage` | 开机播报内容 | "您的小龙虾已上线" |

### Step 3：重启网关并测试

重启 OpenClaw Gateway：

```bash
openclaw gateway restart
```

然后——

🎉 **等待15秒左右，你会听到小爱音箱说**：

**这说明配对成功了！**

## 🎮 进阶玩法（玩得更爽）

### 1. 定制专属身份

修改 `startupMessage`，让你的小爱有个性的自我介绍：

```json
"startupMessage": "主人，我是你的AI助理小龙，有啥事尽管吩咐！"
```

### 2. 智能消息确认

开启 `acknowledgeOnReceive`，小爱收到消息会回复你：

```json
"acknowledgeOnReceive": true,
"receiveMessage": "好的老板！"
```

每次下指令，小爱都会回应，交互感直接拉满。

### 3. 多设备扩展

如果你有多个小爱设备，可以在 `devices` 里加数组：

```json

```

一次接入，全部联动！

### 4. 接入更多AI能力

OpenClaw本身支持多种AI模型（GPT、Claude等），接入小爱后，理论上可以让小爱调用任何OpenClaw支持的AI能力。玩法多多，等你探索！

## ⚠️ 常见问题

**Q：重启后小爱没反应？**
A：检查一下 `passToken` 是否正确，以 V1 开头，复制时别漏了字符。

**Q：设备找不到？**
A：确认音箱在米家APP里能正常控制，设备名称要**完全一致**（区分大小写）。

**Q：配对成功但语音没声音？**
A：检查音箱音量，确保不是静音模式。
