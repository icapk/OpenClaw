# 内容发布SOP

> 运营部 · 内容发布流程手册
> 制定日期：2026-03-21
> 版本：v1.0

---

## 第一部分：平台发布格式要求

### 一、微信公众号（微信公众平台）

#### 1. 标题规范
- **字数限制：** 最多64个字符（约32个汉字）
- **建议长度：** 14~30个字符（14~20字最佳）
- **禁忌：** 避免特殊符号、过度营销词汇（"最"、"第一"等极限词违禁）

#### 2. 封面图要求
- **尺寸：** 900×383 像素（2.35:1 比例）
- **建议分辨率：** 72dpi以上
- **大小：** 不超过10MB
- **格式：** JPG/PNG
- **要点：** 画面简洁、文字少、主体突出，适合小图预览

#### 3. 摘要（封面简介）
- 最多120个字符（约60字）
- 自动截断前54字，建议手动填写，吸引点击
- 如留空，系统会抓取正文前54字

#### 4. 富文本编辑器排版规范

**段落间距**
- 段间距建议 1.75~2.0 倍行高
- 段前/段后间距 0.5em~1em
- 正文字号建议 15~17px（正文常用16px）

**字体规范**
- 正文：微软雅黑 / 思源黑体（苹果用PingFang）
- 注释/来源：12~14px，颜色偏灰
- 重点加粗：适度使用，一篇文章不超过5处

**emoji使用规范**
- 数量：每篇不超过5个
- 位置：段落开头或结尾，不用在正文中间切割阅读节奏
- 风格统一：不要混用不同风格的emoji

**图片排版**
- 图片宽度建议 1080px 宽（适应手机屏幕）
- 图注文字：居中，14px，灰色，放在图片下方
- 避免图片过大导致加载慢

**引导关注**
- 文末引导：建议放"点在看"+"分享"+"关注"组件
- 关注回复关键词设置好

#### 5. 原创声明
- 原创文章可获得更高推荐权重
- 开启原创标识（需满300字以上）

### 二、小红书

#### 1. 标题规范
- **字数限制：** 最多20个字符（含标点）
- **建议长度：** 12~18字
- **技巧：**
  - 前5字要有关键词（影响推荐）
  - 带数字更吸睛（如"5个技巧…"）
  - 带情感词（如"必看"、"宝藏"）
  - 可用🔥💡等emoji辅助（1~3个）

#### 2. 正文格式规范
- **字数限制：** 最多1000字（超出会被折叠）
- **建议结构：**
  1. 开篇痛点/共鸣（1~2句）
  2. 正文分点（用数字序号，每点配图）
  3. 结尾引导互动（收藏/评论/关注）
- **段落：** 每段不超过50字，移动端阅读体验优先
- **换行：** 段与段之间必须换行，空一行更易读
- **禁止：** 长段堆砌，会被折叠

#### 3. 话题标签（Tags）使用技巧
- **数量：** 最多10个话题标签
- **建议：** 3~5个精准话题 + 2~3个流量话题 + 1个品牌话题
- **格式：** #话题名称#（不要加空格）
- **技巧：**
  - 选择有一定热度但不太卷的话题
  - 热门话题放在前3个（前3个影响推荐权重）
  - 结合内容领域选择垂直话题（如#职场干货、#自我提升）
  - 适当蹭热点话题（节假日前后、流行热词）

#### 4. 封面图建议
- **尺寸：** 3:4（竖版 1242×1662 px）或 1:1（方版）
- **竖版3:4为主流，建议优先使用**
- **要点：**
  - 大字标题（封面即内容摘要）
  - 对比强烈、视觉冲击
  - 人物出镜 > 纯文字 > 纯产品图
  - 封面文字不超过8个字
- **图片比例使用建议：**
  - 3:4（竖版）：信息量大、有冲击力
  - 1:1（方版）：教程类、合集类
  - 16:9（横版）：电脑/数码类内容

#### 5. 发布后优化
- 发布时间：早7-9点 / 午12-14点 / 晚20-22点（最佳）
- 发布后30分钟内：引导点赞、评论（互动影响推荐）
- 评论区置顶：放关键词引导二次搜索

## 第二部分：内容发布标准流程SOP

### 流程总览
```
收到文案 → 预处理 → 平台适配 → 发布 → 数据追踪 → 复盘优化
```

### Step 1：收到文案（交接检查）

**核查清单：**
- [ ] 文案已通过审核（有审核记录）
- [ ] 确认发布平台（公众号/小红书/双平台）
- [ ] 确认发布时间（是否有定时要求）
- [ ] 确认配套素材（图片/视频/附件是否齐全）
- [ ] 记录来源（谁发的、什么时候发的）

**如有问题：** 立即退回上游（文案/视频组），不擅自修改发布。

---

### Step 2：预处理

**公众号预处理：**
1. 复制文案到富文本编辑器（建议用壹伴 / 135编辑器等工具辅助）
2. 检查标题是否符合字数规范（≤64字符）
3. 按规范排版（行间距、字号、段落结构）
4. 插入封面图（按900×383尺寸要求）
5. 撰写封面摘要（≤120字符）
6. 开启原创声明（如适用）
7. 预览检查（手机端预览效果）

**小红书预处理：**
1. 撰写标题（≤20字符，前5字含关键词）
2. 整理正文（控制每段字数，分点清晰）
3. 选定封面图（建议用Canva/稿定设计制作）
4. 选定话题标签（3~5个精准+2~3个流量+1个品牌）
5. 预览检查

---

### Step 3：各平台发布要点

**公众号发布：**
1. 登录微信公众号后台（mp.weixin.qq.com）
2. 新建图文消息
3. 粘贴/编辑正文内容
4. 上传封面图（自动裁剪，检查预览）
5. 填写摘要
6. 选择是否原创
7. 群发（每天限群发1次，订阅号24小时内最多4次）
8. 记录发布成功（截图留存）

**小红书发布：**
1. 打开小红书APP或创作者中心网页版
2. 点击"+"发布笔记
3. 上传封面图（注意尺寸比例）
4. 填写标题（贴合内容，含关键词）
5. 粘贴/编辑正文
6. 添加话题标签（按策略选取）
7. @官方账号（视内容而定）
8. 选择同步平台（如有绑定其他平台）
9. 发布
10. 记录发布成功

---

### Step 4：发布后操作

**发布后30分钟内：**
- 公众号：回复后台留言，检查是否有格式问题
- 小红书：浏览评论区，发布1~2条引导评论的回复

**发布后24小时内：**
- 记录阅读量/点赞/收藏/评论数据
- 截图留存关键数据
- 如数据异常低（<100阅读），排查是否限流/违规

**发布后72小时内：**
- 汇总数据反馈给小麦（数据报表）

---

### Step 5：提升阅读量技巧

**公众号：**
1. 引导"点在看"（文末提醒，点赞转藏比影响推送范围）
2. 留言置顶：选一条优质留言置顶，引导读者互动
3. 朋友圈转发激励：内部人员首日转发扩散
4. 关键词回复：设置好关键词自动回复，引导深度阅读
5. 选题时效性：蹭热点话题可显著提升打开率

**小红书：**
1. 笔记灵感：发布前在"笔记灵感"中投稿，可获得流量扶持
2. 互动引导：文末引导"有问题评论区见"、"收藏起来慢慢看"
3. 更新时间：选择流量高峰期（早/午/晚）
4. 前30分钟互动：发布后自己发几条评论（用小号），带热评论区
5. SEO优化：标题和正文中自然嵌入搜索关键词
6. 系列内容：做系列/合集，增加粉丝粘性和回访

## 第三部分：现状摸底 — 需确认问题清单

请小麦转告主人，确认以下前提条件：

### 一、账号状态（需确认）

| 问题 | 说明 |
|------|------|
| 公众号账号是否已有？ | 需确认账号名称、登录方式（个人/企业） |
| 小红书账号是否已有？ | 需确认账号名称，是否完成实名认证 |
| 两个账号是否同一人管理？ | 确认账号归属，避免权限混乱 |
| 账号是否已认证？ | 认证账号有更多功能和数据权限 |

### 二、第三方发布工具（需确认）

| 问题 | 说明 |
|------|------|
| 是否使用第三方发布工具（如壹伴、135编辑器、即客等）？ | 影响排版流程 |
| 是否使用小红书定时发布工具？ | 官方不支持定时，第三方有风险 |
| 是否有内容日历/排期工具？ | 影响SOP中发布时间节点 |
| 是否接入了数据统计工具？ | 影响数据收集环节 |

### 三、内容合规性（需确认）

| 问题 | 说明 |
|------|------|
| 是否有关键词黑名单（不能出现在文案中的词）？ | 避免违规 |
| 图片是否需要审核（水印、版权）？ | 避免图片侵权 |
| 医疗/金融/教育等特殊行业内容是否有额外限制？ | 平台限制较多 |
| 是否已了解平台社区规范（近期有无违规记录）？ | 影响账号权重 |

### 四、发布流程细节（需确认）

| 问题 | 说明 |
|------|------|
| 每篇内容发布前是否需要小麦/主人最终确认？ | 影响SOP中的审批节点 |
| 是否需要留存发布记录（发布日志）？ | 需建立台账 |
| 内容来源是内部创作还是转载/约稿？ | 影响标注方式 |
| 定时发布的时间段是否有规定？ | 影响排期策略 |

### 五、数据追踪（需确认）

| 问题 | 说明 |
|------|------|
| 数据汇报周期是多久？（每日/每周/每月） | 影响复盘节奏 |
| 核心考核指标是什么？（阅读量/点赞/转发/涨粉） | 影响优化重点 |
| 是否需要对比历史数据？ | 需建立数据基准 |
