### 3.2 各技能学习路径

#### 技能1：市场调研与行业分析

**学习目标：** 能够独立完成细分市场的全景分析

**学习路径：**
```
Week 1-2：入门
  → 用AI工具辅助，尝试3次不同行业的市场概况生成
  → 对比AI输出与真实行业报告，找差距
  → 学习用AI做信息聚合（Web Search + AI总结）

Week 3-4：实践
  → 选择一个感兴趣的行业，完成1份完整市场分析
  → 使用AI处理：行业规模数据、竞争格局、政策环境
  → 验证AI在哪些环节提效显著

Week 5-8：深化
  → 学习搭建市场调研框架（波特五力、SWOT、PEST）
  → 用AI验证框架产出的完整性
  → 产出一份可复用的"AI辅助市场调研SOP"
```

**推荐资源：**
- 《精益创业》— 理解市场验证思维
- 艾瑞咨询/36Kr/虎嗅 — 行业报告来源
- AI辅助：MiniMax + Web Search组合使用

## 四、日常探索任务设计

### 4.1 每周AI探索日历

**周一：市场雷达**
- 用AI抓取本周3个目标行业的动态
- 输出：行业周报（200字摘要）
- 消耗：约15-20次调用

**周二：竞品解剖**
- 选取1个竞品/产品
- 用AI分析：功能对比/UI体验/用户评价
- 输出：竞品分析报告
- 消耗：约20-30次调用

**周三：创意发散日**
- 给定一个产品主题
- 用AI进行功能/场景/卖点的大胆发散
- 输出：创意清单（≥10个）
- 消耗：约10-15次调用

**周四：方案撰写**
- 选取一个真实/模拟需求
- 用AI辅助撰写PRD/产品方案
- 对比：纯人工写 vs AI辅助写 的效率和质量
- 消耗：约30-40次调用

**周五：能力验证与优化**
- 专项测试：本周遇到AI"不给力"的场景
- 优化Prompt，提升输出质量
- 消耗：约10-20次调用

### 4.2 月度深挖任务

| 月份 | 主题 | 目标产出 |
|------|------|----------|
| 第1月 | AI产品经理入门 | 完成能力矩阵自查+学习路径图 |
| 第2月 | AI调研工具链搭建 | 搭建"AI辅助市场调研"SOP |
| 第3月 | AI产品设计实战 | 完成1个AI功能的完整设计方案 |
| 第4月 | AI工作流固化 | 将验证有效的人机协作流程文档化 |
| 第5-6月 | 综合产出 | 产出一份可展示的作品集/案例 |

### 4.3 AI辅助竞品分析 SOP

```
Step 1：确定分析对象（选择1个直接竞品）
Step 2：用AI搜集公开信息（新闻/产品介绍/用户评价）
Step 3：用AI做功能对比（列表化）
Step 4：用AI分析用户痛点（从评价中提取）
Step 5：输出结构化竞品报告
  - 产品定位
  - 核心功能对比表
  - 差异化机会点
  - 可借鉴的设计
```

## 五、OPC内容创作探索

### 5.1 AI辅助内容创作场景矩阵

| 内容类型 | AI介入环节 | 提效比例 | 注意事项 |
|----------|------------|----------|----------|
| 选题策划 | 头脑风暴/趋势分析 | 70%↑ | 需人工筛选有价值的选题 |
| 标题撰写 | 生成多个候选+人工精选 | 50%↑ | AI擅长SEO标题，爆款标题需人工判断 |
| 正文撰写 | 大纲→初稿→润色 | 60%↑ | 需人工补充真实经历/案例 |
| 配图建议 | 图片构思/Alt文字 | 40%↑ | 实际图片需人工审核 |
| 数据收集 | 整理/核实数字 | 80%↑ | 关键数据必须人工核实 |
| 评论互动 | 回复草稿生成 | 50%↑ | 需人工确认语气和立场 |

### 5.2 小红书运营AI玩法

**起号阶段（1-30天）：**
```
→ 用AI分析目标赛道爆款笔记的规律
  - 标题关键词
  - 首图风格
  - 正文结构
  - 发布时间规律

→ 用AI生成10个备选选题
→ 用AI写出5个版本的标题
→ 人工选择后发布

每周消耗：约50-80次调用
```

**内容生产阶段（31-90天）：**
```
→ 建立选题库（AI辅助扩展）
→ 建立标题库（AI批量生成，人工精选）
→ 建立正文模板（AI辅助但保持个人风格）
→ 建立爆款复刻SOP

每周消耗：约100-150次调用
```

### 5.3 公众号运营AI玩法

**差异化策略：**
- 公众号比小红书更偏深度，适合输出产品分析/方法论
- AI辅助生产：行业周报、竞品观察、产品思考

**AI辅助写作流程：**
```
1. 选题确定（人工）
2. 资料搜集（AI辅助Web Search）
3. 大纲生成（AI + 人工调整）
4. 初稿撰写（AI草稿 + 人工改写）
5. 润色优化（AI辅助语言精炼）
6. 人工最终审核（防AI味/防错误）
```

### 5.4 爆款内容拆解与复刻

**拆解维度：**
```
爆款因子拆解表
├── 标题：情绪词/数字/悬念/对比
├── 封面：色彩/构图/文字
├── 开篇：前3秒钩子
├── 正文结构：故事线/数据线/情感线
├── 结尾：互动引导/转发激励
└── 评论区：引发讨论的话题点
```

**复刻SOP：**
```
1. 选一个近期爆款（点赞>1000）
2. 用AI分析其爆款因子（输出结构化表格）
3. 提取可复用的内容框架
4. 用AI将框架套用到自己的领域/产品
5. 人工加入个人经历/独特视角
6. 发布并观察数据反馈
```

## 六、600次/5小时的调用分配策略

### 6.1 预算分配建议

**日均预算：** 约120次/天（5小时周期内）
**周预算：** 约600次/周

| 使用场景 | 占比 | 每次消耗 | 每日次数 | 说明 |
|----------|------|----------|----------|------|
| 产品方案撰写 | 35% | 3-5次/任务 | ~25次 | 核心提效场景 |
| 市场/竞品调研 | 25% | 5-10次/任务 | ~15次 | 深度分析消耗多 |
| 内容创作 | 20% | 2-3次/任务 | ~20次 | 日常更新 |
| 学习与验证 | 15% | 1-3次/次 | ~10次 | 能力探索 |
| 备用/应急 | 5% | 不定 | ~5次 | 特殊需求 |

### 6.2 调用效率技巧

```
✅ 高效做法：
1. 一个会话内完成相关任务（利用上下文记忆）
2. 一次生成多个版本，人工选择最优
3. 用结构化prompt减少反复修改
4. 建立个人prompt模板库，减少每次重写

❌ 浪费做法：
1. 每句话单独发一次调用
2. 不明确需求导致多次无效对话
3. 忘记复用已验证有效的prompt
4. 长文本不用摘要直接处理
```
