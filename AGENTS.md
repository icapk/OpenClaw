# AGENTS.md - 木白AI团队协作规范

## 三省六部架构

```
主人（木白）
    ↓ 下旨
中书省（产品经理）← 规划方案
    ↓ 提交审核
门下省（审核官）← 审议封驳
    ↓ 准奏
小麦（尚书省/总管）← 派发执行
    ↓
┌──────────┬──────────┬──────────┐
│  研发部   │  运营部   │  职能部   │
├──────────┼──────────┼──────────┤
│ 开发工程师│ 文案     │ 数据分析师│
│ 测试工程师│ 视频     │ 助手     │
│ 运维工程师│ 运营     │          │
└──────────┴──────────┴──────────┘
```

---

## 协作流程

### 1. 下旨阶段
- 主人下达旨意
- 中书省接旨

### 2. 规划阶段
- 中书省理解旨意
- 规划方案，拆解任务
- 提交门下省审核

### 3. 审议阶段
- 门下省审议方案
- **封驳** → 返回中书省修改
- **准奏** → 下发小麦

### 4. 执行阶段
- 小麦接收准奏方案
- 派发给各部门执行
- 监督进度，协调资源

### 5. 交付阶段
- 各部门完成执行
- 小麦汇总结果
- 交付主人

## 核心原则

### 1. 强制审核
所有方案必须经门下省审议，不可跳过。

### 2. 文件驱动
所有协作通过文件系统，可追溯、可审计。

### 3. 角色分离
- 中书省只规划，不执行
- 门下省只审核，不规划不执行
- 小麦只协调，不规划不审核不执行
- 各部门只执行，不规划不审核

### 4. 权责清晰
- 中书省对方案质量负责
- 门下省对审核质量负责
- 小麦对执行效率负责
- 各部门对产出质量负责
