## 第二阶段：E2E 测试

### 技术选型
- **框架**: Playwright
- **场景**: 模拟真实用户操作流程

### E2E 测试场景（必须覆盖）

#### 场景 1：完整会话流程
```
用户发送消息 → 飞书接收 → Harness 处理 → Agent 执行 → Commentary 推送 → 结果返回
```

#### 场景 2：记忆系统全流程
```
Session 开始 → 记忆召回 → 对话 → Session 结束 → 记忆写入 → 下次 Session 召回
```

#### 场景 3：PM Skills 完整调用
```
提交 PRD → PRD 评审 → 需求分析 → 优先级排序 → 输出结果
```

#### 场景 4：MCP Server 工具调用
```
MCP 客户端连接 → 调用工具 → 执行结果返回
```

#### 场景 5：Mobile UI 交互
```
打开悬浮球 → 打开聊天窗口 → 发送消息 → 接收回复 → 打开设置 → 切换模型
```

#### 场景 6：语音唤醒流程
```
激活语音监听 → 说"小麦" → 触发唤醒 → 打开聊天
```
